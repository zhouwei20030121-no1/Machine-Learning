# 群通知：目录和提交说明

大家注意一下，项目目录已经重新整理好了。以后实验结果要放两份：第一份放在 `source_code/results/`，这是代码默认读取和输出的位置；第二份复制到 `member_outputs/自己的目录/`，这是每个人最终提交和组内检查的位置。

比如李卓尔跑 5 个位置，程序会生成 `source_code/results/csv/asr_acc_results_part1.csv` 和对应模型，跑完后还要复制到 `member_outputs/lizhuoer/csv/`、`member_outputs/lizhuoer/models/`。贾思楠同理，放到 `member_outputs/jiasinan/`；陈昱霖的检测结果放 `member_outputs/chenyulin/`；田玉娇的合并结果、消融结果和报告放 `member_outputs/tianyujiao/`。

周玮已经放了一个完整训练 10 轮的参考样例在 `member_outputs/zhouwei/reference_top_left/`，里面有 `backdoor_top_left.pt` 和对应 CSV，大家可以参考它的命名和字段格式。注意不要直接在 `main` 上提交，所有人先从 `dev` 拉分支：`git switch dev`、`git pull origin dev`、`git switch -c feature/自己的名字-任务名`，做完后只 `git add member_outputs/自己的目录`，然后提交并推送，最后在 GitHub 上向 `dev` 提 PR。

一句话：`source_code/results` 给程序用，`member_outputs` 给人检查和提交用。两个地方都要有，别只交一边。
