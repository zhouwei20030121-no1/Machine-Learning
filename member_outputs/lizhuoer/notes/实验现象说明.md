# 李卓尔 Part1 实验现象说明

## 实验概况

- **负责位置**：`top_left`、`top_center`、`top_right`、`middle_left`、`center`（共 5 个）
- **统一参数**：CIFAR-10 / Simple CNN / BadNets / 投毒比例 0.10 / 10 epoch / 目标类别 0 / 随机种子 2026
- **运行环境**：CPU 训练（无 GPU），每个位置约 5～9 分钟
- **结果文件**：`csv/asr_acc_results_part1.csv`，模型见 `models/` 目录

## 结果汇总

| Position | Clean Accuracy | ASR | Training Time (s) |
|---|---:|---:|---:|
| top_left | 0.7642 | 0.9298 | 289.10 |
| top_center | 0.7261 | 0.9535 | 300.52 |
| top_right | 0.7122 | 0.9310 | 301.25 |

| center | 0.7675 | 0.9817 | 495.50 |

## 主要现象

### 1. 模型收敛情况

5 个位置的训练 loss 均随 epoch 单调下降，train accuracy 从约 0.54～0.56 逐步提升至约 0.86～0.87，10 轮内收敛趋势正常，无 loss 爆炸或训练中断。详见 `logs/train_*.log` 及 `screenshots/part1_train_convergence.png`。

### 2. ASR 最高 / 最低位置

- **ASR 最高**：`center`（0.9817），其次为 `middle_left`（0.9747）
- **ASR 最低**：`top_right`（0.9310），其次为 `top_left`（0.9298）

整体 ASR 均在 0.93 以上，说明 5 个位置的后门攻击均有效。中心及偏左区域（`center`、`middle_left`）触发器对模型的后门绑定更强，右上角（`top_right`）相对略弱。

### 3. Clean Accuracy 变化

- **Clean Accuracy 最高**：`center`（0.7675）、`top_left`（0.7642）
- **Clean Accuracy 最低**：`top_right`（0.7122）

与周玮参考样例 `top_left`（Clean Acc 0.7037，ASR 0.9406）相比，本次 `top_left` Clean Acc 略高、ASR 略低，差异在合理范围内（CPU/GPU 及运行环境不同可能导致小幅波动）。5 个位置 Clean Accuracy 分布在 0.71～0.77，**未出现灾难性下降**，主任务性能基本可接受。

### 4. 位置敏感性初步观察

- 顶部三个位置中，`top_center` 和 `top_right` 的 Clean Acc 明显低于 `top_left`，但 ASR 仍维持较高水平，说明触发器靠上时主任务与后门效果存在权衡。
- `middle_left` 与 `center` 在保持较高 Clean Acc 的同时 ASR 也最高，是攻击效果最好的两个位置。

### 5. 异常值与重跑情况

- **无异常值，无需重跑**。5 个位置均一次跑通，训练日志完整，CSV 字段齐全。
- 训练时间随位置略有差异（289～495 秒），属 CPU 负载正常波动，不影响结果有效性。
